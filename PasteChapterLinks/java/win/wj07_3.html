7-17 バリデーション処理を実装する

フォームへの入力内容については、「金額だから数字だろう」という考えは捨てましょう。ユーザーが意図せず、もしくはわざと想定外の入力を行うかもしれません。それを防ぐために、 7-1-3 で設計した通りのチェック機能を実装します。

クライアント側（ブラウザ）でチェックした内容と、ほぼ同じかそれ以上のチェックをサーバー側で行っていますが、なぜ同じことを2回も確認するのでしょうか？それは、「Javascriptが動かないかもしれない／ユーザーが意図的に無効にしているかもしれない」とか、「悪意のある第三者がブラウザ以外の方法でリクエストを送信してくるかもしれない」とか、「途中で書き変わっているかもしれない」など、サーバー側で確実にチェック済みのデータを受け取れる保証が無いからです。また、「必須かどうか」ぐらいのチェックの為にわざわざサーバーと通信していてはユーザーのストレスに繋がってしまうという観点からも、必ずクライアント側とサーバー側両方でチェックを行いましょう。

1.

まずはサーバー側のバリデーション処理を設定しましょう。7-16-1で作成したBalanceDataFormの各項目にアノテーションを追加します。 7-1-3 のチェック内容と照らし合わせてみましょう。

＜ポイント＞

(1)

occurredAtは日付ですが文字列として扱い、必須チェックと日付有効性チェックは、別途メソッドを用意してチェックしています。チェック内容としては単純に「yyyy-mm-dd」形式で日付型への変換を試みて、エラーならば日付として無効であると判定しています。「@AssertTrue」の使い方については、 7-8-1 を確認してください。

(2)

「@Size」は桁数に収まっていなければエラーとなるため、必須チェックも兼ねています。

(3)

typeのチェック内容については、本来「収入」か「支出」以外をエラーとするのが正しい仕様です。「@Pattern」というアノテーションで可能となりますので、今後自分で調べてみましょう！

(4)

amountは金額ですが文字列として扱います。「@Digits」によって数値用のチェックが行えるため、それを活用しています。「integer」が整数部の桁数、「fraction」が小数部の桁数ですので、この設定で1～8桁の数値であることを強制できます。ただし、これでは0も許可されてしまいますので、それをエラーとする実装は、今後自分でやってみましょう！

package net.digskill.myapp.form;

import java.time.LocalDate;[7]


import java.time.format.DateTimeFormatter;


import java.time.format.DateTimeParseException;


import java.time.format.ResolverStyle;


import javax.validation.constraints.AssertTrue;


import javax.validation.constraints.Digits;


import javax.validation.constraints.Size;


import lombok.Data;


import lombok.NoArgsConstructor;



@Data


@NoArgsConstructor


public class BalanceDataForm {



    private Long id;



    private String occurredAt;



    @Size(min = 1, max = 20, message = "タイトルは必ず20文字以下で指定してください。")


    private String title;



    @Size(min = 2, max = 2, message = "区分は必ず2文字で指定してください。")


    private String type;



    @Digits(integer = 8, fraction = 0,


            message = "金額は必ず1桁から8桁の間で指定してください。")


    private String amount;



    private String description;



    @AssertTrue(message = "日付は必ず有効な日付を指定してください。")


    public boolean isOccurredAtValid() {


        boolean isValid = true;


        DateTimeFormatter df = DateTimeFormatter.ofPattern("uuuu-MM-dd");


        try {


            LocalDate.parse(this .occurredAt, df.withResolverStyle(ResolverStyle.STRICT));


        } catch (DateTimeParseException e) {


            isValid = false;


        }


        return isValid;


    }


}

一覧を表示させているlist.htmlでは、BalanceDataがDBから取得して保持しているデータをget～メソッドによってそのまま取得して表示させているだけです。View側での加工は好ましくないため、BalanceData側で加工してあげるようにしましょう。

＜ポイント＞

(1)

日付は「2019-12-31」の形式をLocalDate型に変換して、それを「2019年12月31日(火)」の書式に変換して返却しています。

(2)

金額はNumberFormatクラスによって「日本の通貨形式」に書式変換して返却しています。

(3)

入力フォームでメモを入力する際にEnterキーで改行できますが、実は目に見えない「改行文字」を入力しているのです。この改行文字は環境により「キャリッジリターン」や「ラインフィード」のいずれか、もしくは両方で構成され、それぞれ正規表現で「\r」「\n」になります。これをHTML上の改行である「<br>」タグに置換して返却しています。

サンプルコード：BalanceData.java（一部追加）

    // 日付書式変換用フォーマッター


    private static final DateTimeFormatter FORMAT_OCCURRED_AT =


            DateTimeFormatter.ofPattern("uuuu-MM-dd" , Locale .JAPAN);


    private static final DateTimeFormatter FORMAT_OCCURRED_AT_FORMATTED =


            DateTimeFormatter.ofPattern("uuuu年MM月dd日(E)" , Locale .JAPAN);


    // 金額書式変換用フォーマッター


    private static final NumberFormat FORMAT_AMOUNT_FORMATTED =


            NumberFormat.getCurrencyInstance(Locale .JAPAN );


    // 改行文字変換用文字列


    private static final String NEW_LINE_ALL = "\\r\\n| \\n|\r";


    private static final String HTML_BR = "<br>";



    public boolean isPlus() {


        return this.type. equals ( Type.PLUS);


    }



    public String getOccurredAtFormatted() {


        LocalDate date = LocalDate.parse (this .getOccurredAt(), FORMAT_OCCURRED_AT);


        return date.format(FORMAT_OCCURRED_AT_FORMATTED);


    }



    public String getAmountFormatted() {


        return FORMAT_AMOUNT_FORMATTED.format( this .getAmount());


    }



    public String getDescriptionFormatted() {


        return this.getDescription(). replaceAll (NEW_LINE_ALL, HTML_BR);


    }


2.

加工した書式で表示させるため、list.htmlを変更しましょう。

＜ポイント＞

(1)

日付は単純に書式変換を行うgetメソッドを呼び出すように変更しています。

(2)

金額はgetメソッドの変更と、併せて「収入なら緑」、「支出なら赤」の色分けを行っています。

(3)

メモは「<br>」というHTMLタグを含む可能性がありますが、「th:text」ではHTMLタグをそのまま文字列として表示されるように「エスケープ」処理が行われてしまいます。文字列の「<br>」ではなく改行して表示できるよう、「th:utext」を使用して書式変換後のgetメソッドの結果を表示しています。

サンプルコード：list.html（日付表示部分）

<td class="text-center" th:text ="${data.getOccurredAtFormatted()}"></td>


サンプルコード：list.html（金額表示部分）

<td class="text-right">


    <div th:class="'font-weight-bold'


            + (${data.isPlus()} ? ' text-success' : ' text-danger')"


            th:text="${data.getAmountFormatted()}"></ div>


</td>


サンプルコード：list.html（メモ表示部分）

<td class="text-left" th:utext = "${data.getDescriptionFormatted()}"></td>
