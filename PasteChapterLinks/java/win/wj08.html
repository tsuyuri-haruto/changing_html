8-7 HEROKUへ家計簿アプリをデプロイ（展開）する

作成した家計簿アプリに他の人からもアクセスできるようにして、自分のポートフォリオとして公開しましょう！そのためにはアプリケーションを配置するサーバーが必要ですが、サーバーを借りてWEB／AP／DBサーバーの各機能を構築するには、もっと幅広い知識や技術が必要となり、時間やお金もかかります。

そこでHEROKUというサービスを利用しましょう。HEROKUは自分専用のサーバー領域を無料で借りることができ、さらにいくつかのコマンドを実行するだけで、環境の構築とアプリケーションの配置までを簡単に行えるサービスです。

以下の手順で進めていきます。

8-1. HEROKUで使用できるRDBMS、PostgreSQLのローカル環境をdocker上に構築

8-2. SpringBootでPostgreSQLを使用するために設定を変更・追加

8-3. PostgreSQL向けにSQLファイルを変更

8-4. Javaソースコードの内容を一部変更

8-5. ローカル環境でPostgreSQLの動作確認

8-6. HEROKUサービスにサインアップ、クライアントツールインストール

8-7. HEROKU CLIによる設定とHEROKUサーバーへのデプロイ（展開）

8-1 PostgreSQLをdocker上に構築する

HEROKUではPostgreSQLというデータベースを無料で利用することが可能です。まずはdocker上にPostgreSQL環境を構築して、ローカル環境でデータベースを切り替えてもアプリケーションが稼動することを確認しましょう。

1.

MySQLを起動するために 2章 でダウンロードして利用していたdocker-compose.ymlへ、PostgreSQLを起動するための設定を追加しましょう

サンプルコード：Desktop/workspace/docker/compose/docker-compose.yml（追記）

  db2:


    image: postgres:10-alpine


    ports:


      - 5432:5432


    environment:


      POSTGRES_DB: mydb


      POSTGRES_USER: mydb_user


      POSTGRES_PASSWORD: mydb_password


    volumes:


      - ./postgres:/var/lib/postgresql


2.

dockerコンテナが起動状態の場合にはdocker-compose downコマンドによりコンテナを終了させてください。

3.

docker-compose up -dコマンドにより、MySQLに加え、PostgreSQLのコンテナも起動しましょう。

4.

docker-compose psコマンドにより、dbとdb2の両コンテナが正常に起動していることを確認してください。

8-2 SpringBootへPostgreSQL用の設定を追加する

PostgreSQLをSpringBootから使用できるようにいくつかの設定を変更・追加しましょう。

1.

SpringBootからPostgreSQLへ接続するためのドライバを依存関係に設定します。プロジェクトフォルダのルート階層にあるbuild.gradleのdependenciesに以下の通り1行を追加してください。

サンプルコード：build.gradle（追記）

dependencies {



（省略）



    runtimeOnly 'org.postgresql:postgresql'


}


2.

SpringBootの設定ファイルapplication.ymlの中にある、DB接続情報を以下の通り変更してください。その際、変更前の設定は消さずに「//」でコメントアウトしておきましょう。

サンプルコード：src/main/resources/application.yml（変更）

spring:


  datasource:


    url: jdbc:postgresql://192.168.99.100/mydb


    username: mydb_user


    password: mydb_password


    driver-class-name: org.postgresql.Driver


3.

PostgreSQLを使用する場合に必要な設定ファイルを、application.ymlと同じ階層に追加してください。

サンプルコード：src/main/resources/hibernate.properties（新規作成）

hibernate.jdbc.lob.non_contextual_creation=true


8-3 MySQL用のSQLをPostgreSQL向けに変更する

MySQLとPostgreSQLでは使用できるSQLに若干違いがありますので、マイグレーション用のSQLを変更します。

1.

usersテーブル作成用のSQLを変更します。MySQL用のSQLはコメントアウトで残しておき、次のSQLをファイルの後方に追記するようにしましょう。大きな違いは、idをAUTO_INCREMENTの代わりにSERIAL型としている点と、PRIMARY KEYの指定方法が異なる点です。これらの変更は、MySQLとPostgreSQLの違いによるものであるため、毎回必要な変更点となります。

サンプルコード：src/main/resources/db/V1__create_table_users.sql（変更）

DROP TABLE IF EXISTS users;


CREATE TABLE IF NOT EXISTS users (


    id SERIAL NOT NULL,


    username varchar(40) NOT NULL,


    displayname varchar(10) NOT NULL,


    password varchar(64) NOT NULL,


    PRIMARY KEY (id)


);


2.

続いてbalance_dataテーブル作成用のSQLを変更します。usersと同様にMySQL用のSQLはコメントアウトで残しておき、次のSQLをファイルの後方に追記してください。大きな違いは、occurred_atをdate型からvarchar(10)型に変更している点と、activeをboolean型からinteger型に変更している点です。

（これらの変更は、MySQLとPostgreSQLおよびSpringBoot(JPA)の「型の扱いの厳密さ」の違いによるものであるため、最初から考慮しておけば本来は必要の無い変更です。）

サンプルコード：src/main/resources/db/V2__create_table_balance_data.sql（変更）

DROP TABLE IF EXISTS balance_data;


CREATE TABLE IF NOT EXISTS balance_data (


    id SERIAL NOT NULL,


    user_id integer NOT NULL,


    occurred_at varchar(10) NOT NULL,


    title varchar(20) NOT NULL,


    type varchar(2) NOT NULL,


    amount integer NOT NULL,


    description varchar(255),


    created_at timestamp NOT NULL DEFAULT CURRENT_TIMESTAMP,


    updated_at timestamp NOT NULL DEFAULT CURRENT_TIMESTAMP,


    active integer NULL DEFAULT 0,


    PRIMARY KEY (id)


);


8-4 JavaのソースコードをPostgreSQLとHEROKU向けに修正する

SQLの変更内容に合わせた修正と、HEROKUで正常に動作させるための修正をJavaに加えます。これらの変更も、最初から考慮しておけば必要の無い変更となります。

1.

balance_dataテーブルのactive列の型をint型に変更したことへの対応を行います。まずはentity内の型をbooleanからintへ変更しましょう。

サンプルコード：BalanceData.java（変更）

    @Column(nullable = true)


    private int active;


2.

前項の変更に合わせ、コンストラクタの呼び出し箇所の引数をtrueから1へ変更しましょう。コンストラクタ自体は、Lombokのアノテーションでコンパイル時に生成されますので、手作業での変更は発生しません。

サンプルコード：BalanceDataService.java（変更）

    public BalanceData addData(User user, String occurredAt,


            String title, String type, int amount, String description) {


        Timestamp now = new Timestamp(System.currentTimeMillis());


        return repository.save(new BalanceData(null, user.getId(),


                occurredAt, title, type, amount, description, now, now, 1));


    }


3.

active項目の状態を変更して論理削除を行っている箇所の引数をfalseから0に変更しましょう。

サンプルコード：BalanceDataService.java（変更）

    public BalanceData deleteData(Long id) {


        BalanceData balanceData = this.getData(id);


        if (balanceData != null) {


            balanceData.setActive(0);


            balanceData = this.repository.save(balanceData);


        }


        return balanceData;


    }


4.

次の変更は、HEROKUサーバーのタイムゾーンに左右されず、日本語で曜日表示するための変更です。ofPatternの引数にLocale.JAPANを追記しています。これは常に指定するように心がけましょう。

サンプルコード：BalanceData.java（変更）

    private static final DateTimeFormatter FORMAT_OCCURRED_AT =


            DateTimeFormatter.ofPattern("uuuu-MM-dd", Locale.JAPAN);


    private static final DateTimeFormatter FORMAT_OCCURRED_AT_FORMATTED =


            DateTimeFormatter.ofPattern("uuuu年MM月dd日(E)" , Locale.JAPAN);


8-5 ローカル環境でのPostgreSQLの動作確認

docker上でのPostgreSQLコンテナの起動とアプリケーション側の修正が完了しましたので、まずはローカル環境での動作確認を行います。

1.

eclipseからmyappを起動しましょう。PostgreSQLにはまだテーブルを作成していませんので、flywayによるマイグレーションが成功していることを示す、以下のようなログが出力されていることを確認しましょう。

2.

アプリケーションが起動したら、登録、ログイン、収支情報入力を行い、その結果が正常に一覧やグラフに表示できること、つまり、MySQLからPostgreSQLに変えたことによる差異が何もないことを確認しましょう。

3.

問題が無ければ、 7-11 を参考にしてテストデータの大量登録も確認しておきましょう。

8-6 HEROKUへの登録とクライアントツールのインストール

続いてHEROKUを使用する準備を進めます。HEROKUはサーバー領域やその上で稼働するWEB／AP／DB各サーバー機能の構築や管理まで、すべてWEBサービス上で行うことが可能です。また、作成したアプリケーションをその環境へ配置（デプロイ）する際は、HEROKUのクライアントツールをインストールして、いくつかコマンドを実行するだけで公開が完了となります。

1.

まずはHEROKUアカウント（無料）を作成します。ブラウザで「 https://signup.heroku.com/ 」にアクセス、下図を参考に入力し、最後に「CREATE FREE ACCOUNT」ボタンを押してください。

2.

入力したメールアドレスに認証メールが届きますので、メールの中程にある「https://id.heroku.com/account/accept/～」のリンクを押下してください。

3.

アカウント用にパスワードを入力し、「SET PASSWORD AND LOG IN」ボタンを押してください。「I would like to ～」のチェックは外してしまって構いません。

4.

次にアプリケーションをHEROKUにデプロイするためのHEROKUクライアントツールをインストールします。「 https://devcenter.heroku.com/articles/heroku-cli 」を開き、「64-bit installer」をクリックしましょう。

5.

ダウンロードされたインストーラーを起動すると、次の画面になりますので、すべてチェックして「Next >」をクリックしてください。

6.

ダウンロードされたインストーラーを起動すると、次の画面になりますので、すべてチェックして「Next >」をクリックしてください。

7.

インストールが完了したら、「Close」を押して画面を閉じましょう。

8.

HEROKUクライアントツールをインストールすると、herokuというコマンドが使用可能となります。Docker Quickstart Terminalを起動して、herokuコマンドを実行してみましょう。下記のように表示されれば、インストールは成功です。

※もし「コマンドが見つからない」とのエラーが表示されたら、PCを再起動してください。

9.

herokuコマンドを使って、HEROKUにログインしてみましょう。heroku loginを実行して、Press any key～が表示されたらEnterキーを押してください。ブラウザでHEROKUのログイン認証ページが表示されますので、画面に従ってログインしてください。Logged in as～と表示されればログイン成功です。

8-7 HEROKUへ家計簿アプリをデプロイ（展開）する

HEROKUへ「家計簿アプリ」をデプロイします。まずはHEROKU上に「アプリケーション」を作成して、WEB/APに当たる機能を担うサーバー領域を準備します。次に、DBとしてPostgreSQLが稼動するサーバー領域を「アドオン」として追加します。そして、アプリ側からDBへアクセスできるように接続先等の設定を行えば、後はアップロードして、デプロイ完了となります。

1.

自分のHEROKUアカウント上に「アプリケーション」を作成しましょう。「myapp」フォルダへ移動してから、heroku createを実行してください。「Creating app～」に続く文字列が、作成されたアプリケーションを一意に特定するためのID（アプリケーション名）になります。青文字の「.herokuapp.com」が付いたURLでアクセスが可能となり、緑文字のgitリモートリポジトリにpushすることでデプロイできるようになります。

2.

heroku openコマンドを実行すると、ブラウザが起動されてアプリケーションのURLにアクセスされます。まだデプロイしていないため、デフォルトのWelcome画面が表示されればOKです。これで、あなた専用のWEB/APサーバーが、WEB上に公開されたことになります！

3.

続いて、DBサーバーを「アドオン」します。次のコマンドを実行してPostgreSQLサーバーを作成しましょう。

heroku addons:create heroku-postgresql --version=10

以下のように表示されれば成功です。

4.

作成したDBサーバーへの接続情報を取得し、application.ymlに設定します。heroku configコマンドを実行、取得されたDATABASE_URLを分解して、それぞれ対応する箇所に設定してください。

postgres://【■ユーザー名】:【■パスワード】@【■サーバー名】:【ポート番号】/【■データベース名】

サンプルコード：src/main/resources/application.yml（変更）

    url: jdbc:postgresql://【サーバー名】:【ポート番号】/【データベース名】?sslmode=require


    username: 【ユーザー名】


    password: 【パスワード】


5.

デプロイする際にHEROKUが「SpringBootプロジェクトである」ことを判定してくれ、サーバー領域にJavaの実行環境を構築してくれます。適切なバージョンを構築してくれるように、プロジェクトフォルダのルート階層に以下のファイルを作成してください。

サンプルコード：system.properties（新規作成）

java.runtime.version=11


6.

次のコマンドにより、サーバー領域にこのアプリケーションが稼動する環境のタイムゾーンを設定します。

heroku config:add TZ=Asia/Tokyo

7.

それではデプロイを行います。デプロイはHEROKUのgitへpushすることで開始されますので、次の通り順番にコマンドを実行してください。push後に状況によって3分程度かかる場合があります。

(1) git add .

(2) git commit -m “heroku settings”

(3) git push heroku master

※(2) のコミットメッセージ「heroku settings」は自由に設定して構いません。

8.

heroku openしてアプリケーションにアクセスしてみましょう。ローカルに構築していたアプリケーションと全く同じ動作をすることが確認できれば、HEROKUを使用したポートフォリオの公開が完了です！
