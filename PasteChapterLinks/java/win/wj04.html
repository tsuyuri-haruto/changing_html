ここまでは、ブラウザの画面に文字列を表示させるだけのWEBページでした。ここからは、ブラウザからユーザーの入力を受け取り、その結果を返す簡単なWEBアプリケーションを作ってみましょう！

4-1 入力フォームの作成

1.

今まで作ってきた「IndexController.java」では、画面に表示させたい文字をそのままやり取りしていましたが、ネットショッピングで物を買う際の入力フォームなど、様々な入力項目があるフォームを「.append()」だけで組み立てていくのはとても大変です。「見た目」を制御できるHTMLと、条件に応じてそのHTMLを変更できるJavaのプログラムを両方とも記述できる「JSP（Java Server Pages） 」という仕組みを活用することで、効率的にフォームを作ることが可能となります。

「パッケージ・エクスプローラー」パネルで「HelloWorld」プロジェクトを開き、「WebContent」フォルダを右クリックして、「新規」⇒「その他」の順にクリックしてください。

2.

次の画面が開きますので、「Web」の中の「JSPファイル」を選択して、「次へ」をクリックしてください。

3.

次の画面では「WebContent」フォルダが選択されていることを確認して、ファイル名に「form.jsp」と入力して「次へ」ボタンをクリックしてください。

4.

次の画面ではテンプレートに「新規JSPファイル（html 5）」を選択して、「完了」をクリックしてください。

5.

「IndexController.java」やその表示結果が表示されていたパネルに「form.jsp」の中身が表示されます。

「<html> ～ </html>」の中身をサンプルのように書き換えて「Ctrl + s」で保存しましょう。保存したら、「パッケージ・エクスプローラー」で「form.jsp」を右クリックして、「実行」の中にある「サーバーで実行」をクリックしてみましょう。このサンプルではまだHTMLのみで構成されていることに注意しましょう。

＜ポイント＞

(1) 属性：

HTML各タグ名に続く「id」「name」「type」・・・などを「属性」といいます。

タグの種類によって指定すべき属性は異なり、そのタグの機能を補完・拡張します。特にformタグとinputタグについて、id属性はjavascriptでHTMLを操作する場合などに使用し、name属性はサーバー側でフォームの値を認識する際に必要であるため、必ずセットで指定するようにしましょう。

(2) formタグ：

formタグの中にはinputタグなどの入力用フォーム部品を配置し、submitボタンによりその入力内容をaction属性に指定したURLへ送信することができます。

action属性の属性値が空文字「""」の場合には、今のページのURLに送信します。

(3) inputタグ：

inputタグは、type属性によりその見え方や機能を変えることができます。

type属性の属性値が「text」の場合は、文字列（英数字や記号、日本語などすべて）を入力できる入力欄となり、「submit」の場合には、送信を実行するボタンとなります。なお、ボタンの場合はvalue属性の値がボタンに表示されます。

サンプルコード：form.jsp（HTMLタグの中身を置き換え）

    <head>


        <meta charset="UTF-8">


        <title>ごあいさつ</title>


    </head>


    <body>


        <form id="greetings_form" name ="greetings_form" method="POST" action ="">


            お名前を教えてください。<br>


            <input id="your_name" name ="your_name" type="text" value = ""><br>


            今は「朝」？「昼」？「夜」？<br>


            <input id="now" name="now" type="text" value=""><br><br>


            <input id="submit" name= "submit" type="submit" value= "送信！" ><br>


        </form>


    </body>



＜表示結果＞

4-2 フォーム送信データの取得・表示

1.

作成したフォームの送信ボタンを押すと、入力した内容がクリアされるだけに見えると思います。実際には「http://localhost:8080/HelloWorld/form.jsp」へ入力内容を送信していますが、form.jspの中身はHTMLのみで送信データを取得するような処理を行っていないため、最初と同じページを表示させているだけなのです。

次のサンプルのようにJavaコードを書き加えて、送信データを取得して、返すページに表示させてみましょう！

＜ポイント＞

(1) JSPタグ：

「<% ～ %>」の中では、Javaコードを動かすことが可能です。サンプルでは、フォームから送信されたデータを取得して変数に格納したり、文字コードの設定を行っています。

定数・変数や関数の定義などを行う場合は「<%! ～ %>」を使います。

(2) POST：

greetings_formのmethod属性に「POST」と指定しています。これは、HTTP通信のリクエストデータの中にフォームデータを含むことをサーバー側に知らせています。

サンプルでは「your_name=太郎&now=朝」のようなデータ（実際には日本語がHTMLエンコードされています）が送られています。

(3) GET：

ブラウザのアドレスバーにURLを打ち込んだり、リンクをクリックしてアクセスした場合には、HTTP通信の際にGETというメソッドが使用されます。リクエストデータの中にデータを持つことはできませんが、URLの後ろにクエリストリングを付加することでデータを送信することが可能です。

例）http://localhost:8080/HelloWorld/form.jsp?your_name=太郎&now=朝

(4) データ取得：

最初のJSPタグの中に「request」という変数が突然登場します。IndexController.javaのdoGetで引数として受け取っていた「HttpServletRequest request」と同じものが、jspでは暗黙的に用意されますので、ここから「getParameter()」で取得しています。

サンプルコード：form.jsp（BODYタグの中身を置き換え）

    <body>


<%


    request.setCharacterEncoding("UTF-8" ); [1]



    String yourName = request . getParameter("your_name");[2]


    if (yourName == null) {


        yourName = "";


    }


    String now = request.getParameter("now");


    if (now == null) {


        now = "";


    }


%>


        <form id="greetings_form" name ="greetings_form" method="POST" action ="">


            お名前を教えてください。<br>


            <input id="your_name" name ="your_name" type="text" value= "<%= yourName %>"><br> [3]


            今は「朝」？「昼」？「夜」？<br>


            <input id="now" name="now" type="text" value="<%= now %>" >< br><br>[4]


            <input id="submit" name= "submit" type="submit" value= "送信！" ><br>


        </form>


        あなたのお名前は「<%= yourName %>」さんですね！[5]


    </body>



[1] 取得データの文字コードを設定

[2] POSTされたフォームデータを変数へ格納。パラメーター無しならば「null」という「何もないことを示す特殊な値」となるため、それを判定して、空文字「""」に置き換える。

[3] 変数の中身をvalue属性に設定

[4] 変数の中身をvalue属性に設定

[5] <%= 変数名 %>で変数の中身を文字列に変換

「request.getParameter()」でフォームからデータを取得して、表示させることに成功しました。しかし、実はこのコードはとても危険です。名前入力欄に「<script>alert('意図しないjavascriptの実行');</script>」と入力して送信してみてください。

3.

画面上にアラートメッセージが表示されました。これは、「あなたのお名前は～」の部分にyour_nameへ入力した内容がそのまま出力され、ブラウザにより「<script> ～ </script>」の中身がjavascriptとして実行されてしまったことを示します。これがもし、悪意のあるＡさんがこれを入力し、Ｂさんが別のページでこの内容を表示できるシステムだった場合、Ａさんが勝手にＢさんのブラウザで勝手にjavascriptを動かせることになってしまいます。このリスクを「クロスサイトスクリプティング（XSS）」と言います。

＜表示結果＞

4.

また、HTMLタグがそのまま出力されるということは、ユーザーが顔文字のつもりで入力した文字列により、レイアウトが崩れたり、ページの機能が損なわれるなどの結果を招きます。これを防ぐため、htmlspecialcharsという関数で、HTMLタグを安全な文字列にエスケープしましょう。ただし、そもそも入力内容としてどんな文字を許可するかを設計時点であらかじめ決めておき、その範囲内に限定する処理を行うことが大切です。

サンプルコード：form.jsp（JSPタグとformタグの間に追記）

%>


<%!


    private static String htmlspecialchars (String str) {[6]


        return str.replace("<", "&lt;" ).replace(">", "&gt;");


    }


%>


        <form id="greetings_form" name ="greetings_form" method="POST" action ="">



[6] ここを追加


サンプルコード：form.jsp（最後の行を変更）

        あなたのお名前は「<%= htmlspecialchars(yourName) %>」さんですね！


5.

これでアラートメッセージが表示されず、「あなたのお名前は～」に入力内容がそのまま表示されました。

「HTMLタグを安全な文字列にエスケープ」の結果どうなったのか、表示されたページ内のどこかを右クリックして、「ソースを表示」してみましょう。すると、「<script>」が「&lt;script&gt;」に変換されていることがわかると思います。このように、「<」を「&lt;」、「>」を「&gt;」のように「エンティティ化 」することで、ブラウザにHTMLとは認識させず、表示のみ元の文字にさせることが可能です。
