5-3 クラス・プロパティ・メソッド

1.

＜ポイント＞

(1) クラス：

「class クラス名 { ～ }」でくくると、その中がクラスとして扱われます。クラスの中にはこれまでと同様に、定数や変数、関数を持つことができます。変数はクラスのプロパティ、関数はクラスのメソッドとも言います。

(2) インスタンス：

クラスはあくまで設計書であり、その設計書に基づいて実体を作成（実体化）することで、初めてそのクラスに定義されたプロパティやメソッドを使用できるようになります。

実体化はコンストラクタというそのクラスの実体が返り値となる特別なメソッドによって行います。一般的に実体は「インスタンス」と呼ばれます。

(3) アクセス修飾子：

定数やプロパティ、メソッドをクラスの外から参照したり呼び出したりできるかどうかの公開範囲を、アクセス修飾子により定義します。定数ならば「static final」キーワードの前、プロパティならば型名の前、メソッドならば戻り値の型名の前に指定します。

・public： クラス内外から自由にアクセス可

・protected： クラス内部、同一フォルダ内のクラス、継承したクラスからアクセス可

・指定なし： クラス内部、同一フォルダ内のクラスからアクセス可

・private： クラス内部からのみアクセス可

※「継承」については次の章で解説します。

(4) プロパティ：

クラス内で定義されたプロパティは、以下のように参照・代入できます。

・クラス外部： インスタンスを格納した変数.now

・クラス内部： this.now

・子クラス： super.now

(5) メソッド：

クラス内で定義されたメソッドは、以下のように呼出できます。

・クラス外部： インスタンスを格納した変数.getGreetMessage()

・クラス内部： this.getGreetMessage()

・子クラス： super.getGreetMessage()

(4) 定数：

クラス内で定義された定数は、以下のように参照することができます。

・クラス内外： Human.KEYWORD_MORNING

サンプルコード：Human.java（すべて置き換え）

package jp.co.digskill;


public class Human {


    protected static final String BR = "<br>";[4]


    protected static final String HR = "<hr>";


    protected static final String KEYWORD_MORNING = "朝";


    protected static final String KEYWORD_AFTERNOON = "昼";


    protected static final String KEYWORD_EVENING = "夜";



    protected String messageMorning = "おはようございます！";[5]


    protected String messageAfternoon = "こんにちは！";


    protected String messageEvening = "こんばんは！";


    protected String messageOther = "朝・昼・夜で答えてください。";


    protected String name = "";



    public Human() {}[6]



    public Human(String name) {[7]


        this.name = this.htmlspecialchars(name);


    }



    public String greet(String yourName, String now) {


        String message = Human.HR;


        message += String.format("私の名前は「%s」です！", this.name) + Human.BR;


        message += String.format("あなたのお名前は「%s」さんですね！",


                                        this.htmlspecialchars(yourName)) + Human.BR;


        message += String.format("今は「%s」ですね！",


                                        this.htmlspecialchars(now)) + Human.BR;


        message += this.getGreetingMessage(now) + Human.BR;[8]


        return message;


    }



    protected String getGreetingMessage(String now) {


        return now.equals(Human.KEYWORD_MORNING) ? this.messageMorning[9]


        : now.equals(Human.KEYWORD_AFTERNOON) ? this.messageAfternoon


        : now.equals(Human.KEYWORD_EVENING) ? this.messageEvening


        : this.messageOther;


    }



    protected String htmlspecialchars(String str) {


        return str.replace("<", "&lt;").replace(">", "&gt;");


    }


}

5-4 継承・オーバーライド

1.

「人間」の設計図を作り、産み出すと同時に名前を付け、メッセージを送って挨拶させる。この仕組みはうまく機能しており、何人でも人間を産み出すことができます。しかし、異なる言葉を話す人間を産み出すためにはどうすればよいでしょうか？例えば、名前と同時に話す言語名を渡して、内部でそれを元に分岐するというのも一つの手です。ここでは、継承という仕組みを使って、「人間」の特徴をすべて引き継ぎ、言語の部分のみ独自の定義を行う新たなクラスを作ってみましょう。5-3-1を参考に「EnglishHuman.java」を作成し、次のサンプルコードで置き換えてください。そして、課題をクリアしてみましょう！

＜ポイント＞

(1) extends：

クラス名の後ろに「extends」キーワードと共に「Human」クラスを指定しています。これにより、Humanクラスの性質をすべて引き継ぐということを示しています。これを「継承」と言います。もし、４行目以降のコードがすべてなければ、Humanクラスと全く同じ振る舞いをするクラスになる、ということです。

継承元 = スーパークラス（親クラス）

継承先 = サブクラス（子クラス）

(2) package：

Javaの世界では、階層化（グループ化）されたフォルダのことをパッケージと呼びます。パッケージはフォルダ名を「.（ドット）」で繋ぐことで階層を表現します。

ルールとして、パッケージ内にクラスを作成する際は、コードの先頭に、「package フォルダ1.フォルダ2.フォルダx」と宣言する必要があります。（FQCN）これまでimportで宣言していた「java.util.List」等の記述は、「javaフォルダ」内の、「utilフォルダ」内の、「Listクラス」をimportする。と読むことができます。

(3) オーバーライド：

Humanクラスに定義している定数やメソッドのうち、一部のみ、しかも同じ名前と引数で、異なる内容の定義を行っています。これを「オーバーライド」と言います。

「getGreetMessage」はオーバーライドしていませんが、その処理に使用される変数は、コンストラクタで代入した変数が使用されていることがわかると思います。

＜課題＞

(1) 「FormController.java」に追記して、３人目の英語を話す人間を産み出しましょう。

(2) 「form_result.jsp」に追記して、３人目の人間にも、挨拶をさせてみましょう。

※ useBeanタグのclassには、EnglishHumanではなくHumanと指定しても問題なく動きます。これは、EnglishHumanもHumanを継承しているからです。試してみましょう！
