2-1 パソコン上にサーバー環境を構築する

1 章 では、プログラムを動かすために必要な環境を一通り見てみました。その内、パソコンとブラウザは既に手元にありますね。ここでは、Docker というアプリケーションを使用して、DB サーバーをパソコン上に構築していきます。

なお、テスト用の WEB サーバーと AP サーバーは、後で別の方法で用意します。

1.

サーバー環境を簡単に構築できる「Docker Toolbox」をダウンロード・インストールしていきます。

ブラウザで「https://github.com/docker/toolbox/releases」を開き、最新バージョンの Assets にある「DockerToolbox-xx.xx.x.exe」（-xx.xx.x はバージョン）をクリックすると、ダウンロードが始まります。

2.

「Docker Toolbox」のダウンロードが完了したら、その exe ファイルを実行してください。

直後に「ユーザーアカウント制御」が出てきたら、「はい」をクリックしてください。

3.

インストール画面が表示されますので、「Help Docker imporve Toolbox.」のチェックを外して、「Next >」をクリックしてください。

4.

次の画面では何も変更せず「Next >」をクリックしてください。

5.

次の画面ではすべての項目にチェックを入れて「Next >」をクリックしてください。

6.

次の画面では「Install VirtualBox with NDIS5 driver[default NDIS6]」にチェックを入れて、「Next >」をクリックしてください。

7.

次の画面ではそのまま「Install」をクリックしてください。

しばらく待つとインストール完了画面が表示されますので、「View Shortcuts in File Explorer」のチェックを外して、「Finish」ボタンを押してください。

インストールされた「VirtualBox」を、最新バージョンにアップデートします。

デスクトップに作成された「Oracle VM VirtualBox」のショートカットを実行してください。

表示されたダイアログの「次のリンクから最新バージョンをダウンロードできます。」に続くリンクをクリックしてください。

10.

「VirtualBox-6.x.xx-xxxxxx-Win.exe」のダウンロードが完了したら、現在開いている「Oracle VM VirtualBox」を閉じてから、その exe ファイルを実行してください。

インストール画面が表示されますので、「Next >」をクリックしてください。

12.

次の画面では何も変更せず「Next >」をクリックしてください。

13.

14.

次の画面ではそのまま「Install」をクリックしてください。

15.

しばらく待つとインストール完了画面が表示されますので、「Finish」ボタンを押してください。

16.

インストールされた最新の「VirtualBox」が起動しますので、そのままウインドウを閉じてください。

17.

「Docker Toolbox」のインストールが完了しましたので、サーバーの構築に移ります。

デスクトップに作成された「Docker Quickstart Terminal」のショートカットを実行してください。

直後に「ユーザーアカウント制御」が出てきたら、「はい」をクリックしてください。

18.

次のような画面が表示され、しばらく待つとクジラ（！？）のアスキーアートが表示されます。

これで、あなたのノート PC 上に Linux という OS で動く仮想サーバーが起動されました。このサーバーを「docker-machine」と呼ぶことにしましょう。Docker Quickstart Terminal では、この docker-machine を起動して、コマンドで操作ができる状態にしてくれます。今後、クジラのすぐ下の行に表示されている「IP アドレス」を使って、docker-machine にアクセスを行いますので、覚えておきましょう。

ノートパソコンと Docker-machine の関係性

19.

ここまでのステップで次のような画面が表示された場合には、「はい、～」をクリックしてください。

20.

ここからは、docker-machine 上にさらに DB サーバーを構築していきます。また、DB を管理するための、phpMyAdmin というアプリケーションを動かすサーバーも一緒に構築します。

そのために必要なフォルダやファイルを置いておくために、デスクトップに「workspace」というフォルダを作成しましょう。そして、そのフォルダを右クリックしてフォルダのパスを確認しておきましょう。

21.

Docker Quickstart Terminal で以下の順にコマンドを入力して、「workspace」フォルダへ移動しましょう。

(1) 「$」マークの右側へ cd /c/Users/ を入力。（cd の後に半角スペース 1 つ）

(2) 先ほど確認したパスの「Users」と「Desktop」の間（パソコンのユーザーアカウント）を入力。

(3) 続けて /Desktop/workspace を入力。

(4) ここまでで hanaco さんなら「cd /c/Users/hanaco/Desktop/workspace」となっているはずです。

(5) Enter キーを押してコマンド実行。

(6) pwd と入力、Enter を押して、表示されたパスが正しいことを確認。

22.

あらかじめ用意してある設定ファイルを、「Git」という仕組みを使ってダウンロードしましょう。

次のコマンドを実行すると下記のようなフォルダ構造が workspace の中に作成されます。

「git clone https://bitbucket.org/digskill_java/docker.git」

(1) docker / compose / docker-compose.yml

(2) docker / compose / mysql / .gitkeep

(3) docker / compose / pma / sessions / .gitkeep

23.

ダウンロードしてきた設定ファイルを使って、docker-machine の上にさらに DB サーバーと DB 管理アプリ用のサーバーを起動します。「cd docker/compose」コマンドにより、「docker-compose.yml」のある compose フォルダへ移動しましょう。

24.

「docker-compose up -d」コマンドにより、「docker-compose.yml」に定義してある情報を元にして、「mysql」という DB アプリケーションが稼動する DB サーバーと、「phpMyAdmin（PMA）」という PHP 製の mysql 管理アプリケーションが稼動する WEB/AP サーバーが、docker-machine 上で起動します。

25.

これでブラウザから「http://2-1-18 で確認した IP:8080/」にアクセスし、次の画面が表示されれば完了です！

もしもうまく表示されなかった場合は、もう一度これまでの手順を見直し、それでもうまくいかなければ、講師に相談してみましょう。（Microsoft Edge の場合は、Google Chrome をインストールしてください。）

2-2 サーバー環境の起動・停止・リセット

docker-machine 上で起動した DB サーバーや PMA 用のサーバーは、「コンテナ」と言います。このコンテナや、docker-machine を停止させ、Docker Quickstart Terminal を終了させる方法を覚えておきましょう。

ノート PC をシャットダウンする場合にはこの手順を実行するようにしてください。また、次に作業を再開する時の手順も併せて確認しておきましょう。

1. ノート PC シャットダウン前の作業

まず、コンテナを「docker-compose stop」で停止させましょう。起動しているか停止しているかわからない時は、「docker-compose ps」で確認できます。（「指定されたパスが・・・」は気にしなくて大丈夫です！）

次に、docker-machine を「docker-machine stop」で停止させましょう。起動しているか停止しているかわからない時は、「docker-machine ls」で確認できます。（ps ではなく ls でなので注意しましょう！）

最後に、Docker Quickstart Terminal を「exit」で終了させましょう。

2. ノート PC 起動後の作業

まず、Docker Quickstart Terminal をショートカットから起動し、クジラを待ちましょう。（2-1-17 参照）

この時点で、docker-machine は起動されています。

次に、「cd /c/Users/xxx/Desktop/workspace/docker/compose/」コマンドにより、「docker-compose.yml」のある compose フォルダへ移動し、コンテナを「docker-compose start」で起動させましょう。

3. コンテナの初期化を行う場合

コンテナに直接行った設定をリセットしたい場合、「docker-compose down」で停止と同時にコンテナを削除します。その後、2-1-24 を参考にして、「docker-compose up -d」でコンテナを起動しましょう。

これでコンテナが 1 から再作成されます。

2-3 パソコン上に統合開発環境を構築する

実際にプログラムを書くにあたっては、「統合開発環境」というものを使用します。もちろん、「メモ帳」だけでも作れないことはありませんが、テスト用の WEB/AP サーバーをボタンひとつで構築できるだけでなく、たくさんのファイルを管理しやすくする機能や、入力を補助する機能など、開発の現場では無くてはならない存在なのです。

1.

統合開発環境の「eclipse」を日本語化した「Pleiades」をダウンロード・インストールしましょう。

ブラウザで「https://mergedoc.osdn.jp/」を開き、「Eclipse xxxx-xx 最新版」をクリックしてください。

2.

開かれた画面で「Java – Windows 64bit – Full Edition」の「Download」をクリックしてください。

3.

開かれた画面で自動的に「pleiades-xxxx-xx-java-win-64bit-jre_xxxxxxxx.zip」ファイルのダウンロードが開始されます。1.6GB 程度あるためしばらく時間がかかります。

4.

ダウンロードされた zip ファイルの中身は長い名前のファイルが多く、かつファイル数も膨大なため、Windows標準の展開（解凍）処理では、失敗したり時間がかかり過ぎたりしてしまいます。「7-Zip」というフリーソフトをインストールして、そのソフトで展開しましょう。「https://sevenzip.osdn.jp/download.html」開いて、「(MSI インストーラ)7-Zip 64 ビット Windows x64(Intel 64 or AMD64)用」の「ダウンロード」をクリックしてください。

5.

次の画面は広告がたくさん出てきて若干不安になりますが、「何もせず」待っていると、下に警告メッセージが表示されますので（Chrome の場合）、「保存」をクリックしてください。

6.

保存された「7z1900-x64.msi」をダブルクリックして、インストーラーを起動してください。

7.

表示された画面では、そのまま「Next」をクリックしてください。

8.

次の画面では、「I accept the ～」にチェックを入れて、「Next」をクリックしてください。

9.

次の画面では、そのまま「Next」をクリックしてください。

10.

次の画面では、そのまま「Install」をクリックしてください。

11.

この際、ユーザーアカウント制御のダイアログが出ますので、「はい」をクリックしてください。

12.

この画面が表示されればインストール完了です。「Finish」をクリックしてください。

13.

「pleiades-xxxx-xx-java-win-64bit-jre_xxxxxxxx.zip」を右クリックし、「7-Zip」の「ここに展開」をクリックしてください。

14.

7-Zip の展開画面が表示されますので、この画面が閉じると展開完了です。デスクトップに「pleiades」というフォルダが作成されますので、今後はこのフォルダを起点に課題を進めていきます。

15.

「...\pleiades\eclipse\eclipse.exe」をダブルクリックして起動してください。eclipse のロゴが表示されてしばらく内部処理が行われますが、これには数分かかることもあります。その途中で「ワークスペースとしてのディレクトリー選択」画面が表示されますので、「../workspace」のまま「起動」をクリックしてください。

この設定により、「...\pleiades\workspace」フォルダの中に作成中のプログラムなどを格納するように設定が行われます。これは毎回表示されますが、そのまま「起動」をクリックして構いません。

16.

次のような画面が表示されたら、統合開発環境「eclipse」のインストールが完了です。

17.

続いて、プログラムを書く準備をしていきましょう。eclipse ではプログラムやその実行環境を「プロジェクト」というフォルダ構造で管理します。メイン画面左上の「パッケージ・エクスプローラー」に表示されている、「プロジェクトの作成」をクリックしてください。

18.

開かれた画面で、「Web」の中の「動的 Web プロジェクト」をクリックし、「次へ」をクリックしてくだい。

19.

開かれた「新規動的 Web プロジェクト」画面では、以下の通り入力して「次へ」をクリックしてください。

(1) プロジェクト名 ：「HelloWorld」

(2) ターゲット・ランタイム ：「Tomcat9 (Java11)」

(3) 動的 web モジュールバージョン ：「4.0」

(4) 構成 ：「Tomcat9 (Java11) デフォルト構成」

※：(3)と(4)は(2)を選択すると自動的に上記の値に変更されます。

次の画面ではそのまま「次へ」をクリックしてください。

次の画面でもそのまま「完了」をクリックしてください。これで「HelloWorld」プロジェクトが作られます。

続いて JAVA プログラムを書く「サーブレット」ファイルを作成します。HelloWorld プロジェクトを開いて、その中の「src」フォルダを右クリックして、「新規」の「その他」をクリックしてください。

23.

開かれた画面で、「Web」の中の「サーブレット」をクリックし、「次へ」をクリックしてください。

24.

開かれた「サーブレット作成」画面では、以下の通り入力して「次へ」をクリックしてください。

(1) Java パッケージ ：「jp.co.digskill」

(2) クラス名 ：「IndexController」

25.

次の画面ではそのまま「次へ」をクリックしてください。

26.

次の画面もそのまま「完了」をクリックしてください。

27.

作成された「HelloWorld/src/jp.co.digskill/IndexController.java」ファイルをダブルクリックで開くと、下図のように右側にファイルの中身が表示されます。

28.

とりあえずこのままでも動きますので、起動してみましょう。IndexController.java を右クリックして、実行の中の「サーバーで実行」をクリックしてください。

29.

この「サーバー」とは、 1-8 の図で言う「ブラウザとの通信を管理する WEB サーバー」と「JAVA プログラムを処理する AP サーバー」の両方の機能を兼ねる「Tomcat」というアプリケーションのことを示しています。以降の手順ではこの Tomcat をノート PC 上に起動して、ノート PC 内で「ブラウザ⇔WEB/AP サーバー」の環境を構築します。開かれた「サーバーで実行」画面では、以下の通り入力して「次へ」をクリックしてください。

(1) サーバーのタイプを選択 ：「Tomcat v9.0 サーバー」

(2) サーバーのホスト名 ：「localhost」

(3) サーバー名 ：「ローカルホスト の Tomcat9 (Java11)」

(4) サーバー・ランタイム環境 ：「Tomcat9 (Java11)」

(5) 「このプロジェクトを実行するときは常にこのサーバーを使用」にチェックを入れる

30.

次の画面ではそのまま「完了」をクリックしてください。

31.

すると、次のような画面となります。左下の「コンソール」ビューには、Tomcat 起動時のログが出力されています。右側半分には内部ブラウザが起動され、「http://localhost:8080/HelloWorld/IndexController」へアクセスしています。その結果、IndexController.java に書かれたプログラムが実行され、「Served at: /HelloWorld」という文字列が表示されたのです。これですべての準備が完了です！
