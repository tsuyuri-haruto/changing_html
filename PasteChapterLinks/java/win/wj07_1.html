7-1 家計簿アプリの仕様

1. 要件定義

(1) 毎日の収入と支出の金額をメモ付きで入力して管理したい。

(2) ユーザー毎に専用の家計簿が利用できるようにしたい。

(3) 収支の情報は変更や削除ができるようにしたい。

(4) 収支の情報を一覧表示したい。

(5) 収支の状況をグラフ表示したい。

＜登録・ログイン・ログアウト＞

(1) アプリ配下のURLにアクセスすると、未ログインならば「ログイン画面」を表示する。

(2) 全画面共通のヘッダーを設置し、未ログインならば「ログイン画面」と「登録画面」へのリンクを配置、ログイン済ならばログアウトリンクのみのメニューをユーザー名で配置する。

(3) ログイン画面では「メールアドレス」「パスワード」によりログインする。

(4) 登録画面では「氏名」「メールアドレス」「パスワード」を登録する。

(5) ログインすると「トップページ」を表示する。

＜トップページ＞

(1) 「トップページ」には「メニュー」と「収支状況グラフ」を設置する。

(2) 「メニュー」には「収支情報編集画面」へのリンクを配置する。

(3) 「収支状況グラフ」では、選択した年月の収支状況をグラフで表示する。

＜収支情報編集画面＞

(1) 「収支情報編集画面」には「収支情報入力フォーム」と「収支情報一覧」を設置する。

3. 詳細設計

＜登録・ログイン・ログアウト＞

(1) URLはそれぞれ、「登録：/register」「ログイン：/login」「ログアウト：/logout」とする。

(2) 「収支情報入力フォーム」では、以下の入力値チェックを行う。

[クライアント側] ※HTML+Javascriptでチェックを行う

[サーバー側] ※Javaの「Bean Validation」の機能でチェックを行う

(3)

「収支情報入力フォーム」は、新規登録モード、変更モード、削除後モードの３つのモードで動作する。

「/balancedata」へのアクセス時は新規登録モードとする。また、フォーム内のクリアボタン押下時はその時のモードによらず新規登録モードでの動作とする。

【新規登録モード】

・ＩＤは空欄かつ入力不可、日付は当日、区分は「収入」を初期値とする。

・送信ボタンにより「/balancedata/create」へPOSTする。

・新規登録に成功した場合、「データの登録が完了しました。」を表示する。その際、フォームの入力内容は破棄せず、新規登録モードとすることで、連続して次の新規登録が行えるようにする。

・新規登録に失敗した場合、「データの登録に失敗しました。再度実行してください。」を表示する。

【変更モード】

・一覧の変更ボタン押下により「/balancedata/edit/{id}」へGETでアクセスした際に、idパラメータに対応するデータをフォームに反映し、ＩＤは入力不可状態とする。

・送信ボタンにより「/balancedata/update/{id}」へPOSTする。

・変更に成功した場合、「データの更新が完了しました。」を表示する。その際、フォームの入力内容（ＩＤを含む）は破棄せず、変更モードとすることで、連続して再変更が行えるようにする。

・新規登録に失敗したら「データの更新に失敗しました。再度実行してください。」を表示する。

【削除後モード】

・新規登録モードの状態で、削除したデータをフォームに反映しておくことで、誤って削除してしまった場合などにそのまま新規登録により再登録が行えるようにする。

(2) 収支情報はテーブル名「balance_data」に管理し、下表の項目でデータを管理する。

早速、Spring Bootに導入した「Spring Security」のログイン機能を使ってみましょう。

import org.springframework.beans.factory.annotation.Autowired;


import org.springframework.context.annotation.Bean;


import org.springframework.security.config.annotation.authentication.builders.AuthenticationManagerBuilder;


import org.springframework.security.config.annotation.web.configuration.EnableWebSecurity;


import org.springframework.security.config.annotation.web.configuration.WebSecurityConfigurerAdapter;


import org.springframework.security.crypto.bcrypt.BCryptPasswordEncoder;


import org.springframework.security.crypto.password.PasswordEncoder;



@EnableWebSecurity


public class SecurityConfig extends WebSecurityConfigurerAdapter {



    @Autowired


    protected void configureAuthenticationManager(AuthenticationManagerBuilder auth) throws Exception {


        auth.inMemoryAuthentication()


            .withUser("testuser")[4]


            .password(this.passwordEncoder().encode("testpass")).roles("USER");


    }



    @Bean


    protected PasswordEncoder passwordEncoder() {


        return new BCryptPasswordEncoder();[5]


    }


}

Spring Securityのデフォルト設定ではログインに成功すると「/」へリダイレクトされますので、この設定を上書きして「/home」へリダイレクトするように、SecurityConfigを以下の通り修正してください。その際、「HttpSecurity」にエラーを示す赤い波線が表示されますので、ポップアップからクイックフィックスの中の「'HttpSecurity'をインポートします（org.springframework.security.config.annotation.web.builders）」をクリックして先頭の「import」文に追加を行いましょう。

サンプルコード：SecurityConfig.java（メソッド追加）

    @Override


    protected void configure(HttpSecurity http) throws Exception {


        http.formLogin().defaultSuccessUrl("/home");


    }

<エラーが起きたら 対処法>

7章では自分でタイピングするコードの量が多くなり、それに伴い、エラーが発生する頻度もより多くなるでしょう。

このコラムではエラーが発生した際の対処法を紹介します。初学者は、大量の英語で書かれたエラーメッセージに戸惑うと思いますが、ポイントを押さえて読み解くことで、自力で解決するスキルを身につけましょう！

(1) エラーのステータスを確認する

サーバーからブラウザに返してくるコードのことをステータスコードといいます。

ex) type=Internal Server Error, status=500

<主なHTTPステータスコード>

200：OK：正常終了

302：Found：リダイレクト

403：Forbidden：禁止（アクセス権限がない）

404：Not Found：ページが見つかりません

500：Internal Server Error：内部サーバエラー（ソースコードに不具合など）

503：Service Unavailable：一時的にサーバーにアクセスできない（過負荷など）

(2) IDEのコンソールログを見に行き、エラー発生個所の先頭を確認する

IDEのコンソールログは、ブラウザに表示されるログよりも詳細な情報が得られることが多々あります。

(3) 自作のフォルダ（パッケージ）およびクラスが記述されたログを、検索を駆使して探し出す

7-8 ユーザー登録画面を作成する

ここまでで、「ログイン」～「トップページ表示」～「収支情報編集画面表示」～「ログアウト」という一連の流れが行えるようになりました。まだログインに使用するユーザー名とパスワードが「ハードコーディング」されている状態となっているため、ユーザー情報をDBに登録し、その情報でログインできるようにします。新たに家計簿アプリを使用するユーザーを登録するための「ユーザー登録画面」を作成しましょう。

1.

仕様を確認すると、登録画面には「氏名」「メールアドレス」「パスワード」および「パスワード（確認用）」の項目があります。氏名は2～10文字程度、メールアドレスは真ん中に「@」があること、パスワードは2回入力したものが一致していること・・・など、入力内容が正しいことを確認し、問題が無い場合のみDBに登録するようにしなくてはなりません。このようなチェックを「入力値検証（Validation:バリデーション）」と言います。

登録画面から届くフォーム入力値を格納し、バリデーションを行うクラスを用意します。「src/main/java」の「net.digskill.myapp」の中に「form」フォルダを作成、その中に「RegisterForm.java」ファイルを作成して、次のコードで置き換えてください。

＜ポイント＞

(1)

メールアドレス＝username、氏名＝displayname、パスワード＝password / confirmPasswordとしてフォームのname属性をクラスのプロパティ名と一致させています。これによって、今後Spring Bootが「DI（7-6-2参照）」する際に自動的にフォームからの送信値を取得することができるようになります。

(2)

Spring Securityのデフォルト機能を使用する場合、「username」と「password」がログイン項目として使用されます。変更する手段はいろいろありますが、このアプリでは極力デフォルト機能を活用するためusernameに日本語を含まず、かつ一意となるようにメールアドレスを設定するようにしました。それに伴い、日本語を含む可能性がある氏名はdisplaynameという別項目にするようにしています。

(3)

@NotBlank、@Email、@Sizeは、付与した項目のバリデーション方式を指定できます。@NotBlankはそのまま「必須」であること、@Emailは「Emailとして正しく認識できる形式」であること、@Sizeは「minとmaxの間」であることを検証し、それ以外の場合はエラーとします。この際、エラーメッセージを「message="～"」として指定することが可能です。なお、confirmPasswordはアノテーションが有効な行の次の行となりますので、@Sizeの検証対象となりません。

(4)

@AssertTrueについては、プロパティではなくメソッドに付与します。このメソッドはbooleanを返却する必要があり、ここでは「パスワードが確認用と一致するか」を検証し、falseの場合にはmessageの内容をエラーメッセージとして表示します。特殊な記法に見えますが、 3-7-6 で紹介した三項演算子を複数連ねているだけです。returnの各行は以下の判定となっています。

[1] もしpasswordがnullでもtrue ⇒ passwordの必須判定は既に@Sizeで実施済み

[2] もしconfirmPasswordがnullならfalse ⇒ confirmPasswordがnullなら一致し得ない

[3] もしpasswordとconfirmPasswordが一致していればtrue

[4] もしpasswordとconfirmPasswordが一致していなければfalse

import org.springframework.beans.factory.annotation.Autowired;


import org.springframework.security.core.userdetails.UserDetailsService;


import org.springframework.security.core.userdetails.UsernameNotFoundException;


import org.springframework.security.crypto.password.PasswordEncoder;


import org.springframework.stereotype.Service;


import net.digskill.myapp.entity.User;


import net.digskill.myapp.repository.UserRepository;



/**

*/

    /**


     * 「users」テーブルの操作を行うリポジトリをDIします。

    private UserRepository repository;

    /**


     * リポジトリからログイン対象ユーザーを検索・取得します。


     *


     * @param username ログイン対象のユーザー名（メールアドレス）


     * @return ログイン対象のUser


     */


    @Override


    public User loadUserByUsername(String username) throws UsernameNotFoundException {


        if (username == null || "".equals(username)) {


            throw new UsernameNotFoundException("Username is empty");


        }


        User user = this.repository.findByUsername(username);


        if (user == null) {


            throw new UsernameNotFoundException("User not found: " + username);


        }


        return user;


    }



    /**


     * UserDetailsServiceインターフェースには定義されていない<br>


     * ユーザー登録用メソッドです。

     * @param password 登録するユーザーのパスワード

    public void registerUser(String username, String displayname, String password) {


        User user = new User(


                null, username, displayname, this.passwordEncoder.encode(password));


        this.repository.save(user);


    }


}
