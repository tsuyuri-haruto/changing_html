1 基礎知識を学ぼう！

あなたはこのテキストや課題によって、「JAVA」で「WEBアプリケーション」を作れるようになります！

とは言っても、「JAVA」や「WEBアプリケーション」とは何なのか、そもそもコンピューターとは・・・？

早くプログラムを書きたいと思いますが、その前に少しだけ、基礎知識をつけておきましょう。

1-1 「コンピューター」とは

目の前にあるノートパソコンやスマートフォンはもちろん、冷蔵庫・洗濯機・掃除機などの家電、ゲーム機、楽器、腕時計、ウォシュレット・・・そして車まで！現代ではあらゆるものにコンピューターが搭載され、暮らしを便利にしてくれています。しかし、その基本はすべて「電気が流れているか、流れていないか（ON / OFF）」だけなのです！

最初は下図のような原始的な計算機であり、ちょっとした計算でも、膨大な時間と電力と場所が必要でした。しかし、技術の進歩によりものすごく小さく、たくさん集積できるようになったことで、高度な計算が可能となったのです。

ボタンのような物理的な部品から、ON / OFFの組み合わせで命令を伝え（入力）、その内容を元に瞬時に高度な計算を行う。そしてその計算結果をまた、コイルのような物理的な部品（電流が流れれば磁石が動く）によってON / OFFを「動き」に変換（出力）することで、画面へ表示したり、車を動かす。これがコンピューターなのです。

電球による原始的な計算機

電球ON： 電球に書かれた数字を1倍

電球OFF： 電球に書かれた数字を0倍

電球1つを1bit（ビット）と言い、4bitあれば、0～15までの16段階（0000 から 1111）の組み合わせを表現できるのです！

1-2 「プログラム」とは

コンピューターに計算を行わせるためには、コンピューターが理解できる0と1の組み合わせで命令を入力すれば良いということが、1-1でわかりましたね。これが「プログラム」なのです。ただし、これでは効率が悪く、「画面に"A"を表示する」だけで途方もない労力が必要となるのでは、コンピューターを使う意味がありません。

そこで、「キーボードの上から４段目、左から２番目のキーが押されたら、画面に"A"を表示する」という一連の処理を行うプログラムを作成することで、人間が普段使う言葉をそのまま入力して、画面に表示させることができるようになりました。このような基本的なプログラムは、「オペレーティングシステム（ＯＳ）」に含まれています。

次に、「ADD」と入力されたら、その前後の数字を加算するというプログラムを用意します。このプログラムは、人間が入力した「ADD」という文字列をＯＳから受け取り、コンピューターの機械語（0と1）に「翻訳＝コンパイル」してくれるのです。ＯＳとコンパイラのおかげで、人間は「10 ADD 5」と入力するだけで、その結果を画面で確認できるようになったのです！このように、コンピューターが理解できるレベルのプログラム言語を低水準言語と言い、JAVAのような人間が理解できる高水準言語は、低水準言語によるプログラムの組み合わせでできています。

プログラム実行の流れ

1-3 「アプリケーション」とは

アプリケーションは普段から頻繁に使っていますよね。「スマホアプリ」の「アプリ」はアプリケーションの略です。メールやブラウザ、ゲームなど、パソコンやスマホの上で動いているものはすべてアプリケーションになります。

コンピューターに思い通りの処理をさせられるという意味では「プログラム」も同じですが、一般的にプログラムと言った場合には、単独の機能や処理を行う命令が書かれたファイルや、それがコンパイルされて実行できる状態のものを示しており、アプリケーションはその集合体（機能を持ったプログラムの塊）を示していると考えてください。

プログラムとアプリケーション

1-4 「WEB」とは

WEBは英語で蜘蛛の巣という意味であり、ITの世界では「網目状のネットワーク」のことを示します。ブラウザのアドレスバーの「http://」の後に「www」が続くサイトが多くみられますが、これは「World Wide Web」のことであり、「世界に広がる網目状のネットワーク」、つまりインターネットのことを示す言葉です。最近はインターネットの代わりに、WEBやネットという言葉が使われます。この仕組みにより世界中のパソコンやサーバーが繋がっています。

蜘蛛の巣 ≒ WEB

1-5 「サーバー」とは

例えばWEB検索をするとき、ブラウザを開いてGoogleなどの検索サイトにアクセスしますよね。この時、あなたのパソコンやスマホは、WEBを通じてGoogleが管理するサーバーへ接続しているのです。

サーバーもコンピューターであり、あなたのパソコンと本質は変わりません。同じようにプログラムが動き、計算を行います。Googleのサーバーであれば、あなたの要求（リクエスト）を受け付け、計算を行い、検索結果を返します。

パソコンとの違いは、その用途に特化して性能を高めたり、無駄な機能（使いやすいデスクトップ画面など）を排除しているかどうかだけです。ある特定の機能を提供・供給（＝serve）するコンピューターがサーバーなのです。

1-6 「ブラウザ」とは

ブラウザはアプリケーションのひとつです。あなたのパソコンには、Internet ExplorerやEdge、Chromeなどが、スマホにはスマホメーカー独自のブラウザや、Chrome、Firefox、Opera、iPhoneならSafariなどがインストールされているかと思います。それぞれ特徴がありますが、基本的には「URLの示すサーバーへアクセスして、HTMLをサーバーから受信して、それを画面に表示させる」機能を持っています。さらに、javascirptというプログラムがブラウザ上で動くようになっていることで、表示だけでなく、より高度な処理を行うことができます。

1-7 「URL」とは

URLは「Uniform Resource Locator」の略で、WEB上のサーバーの住所です。ブラウザのアドレスバーにURLを入力すると、WEB上のサーバーと連携してその住所まで辿り着き、必要なHTMLを受信するのです。

1-8 「WEBアプリケーション」とは

従来のアプリケーションは、まずダウンロードとインストールが必要であり、パソコンやスマホ上ですべての処理が行われ、データの保存先もパソコンやスマホの中で完結していました。

しかし、例えば「買い物をしよう！」と思ったら、ブラウザを開いてそのサイトへアクセスし、何もインストールをすることなくすべての処理が完結します。他にも、メールの送受信やゲームなども、すべてブラウザさえあればできるようになってきました。これが、WEBアプリケーションです。

ではここで、買い物をするときに何が起きているのか、その仕組みを見てみましょう。

WEBアプリケーションの仕組み

1.

ユーザーが「10,000円の赤い靴」の購入ボタンを押します。

2.

ブラウザからWEBサーバーへ、「ユーザーID=AAAAA」「商品ID=BBBBB」「処理＝購入」などの情報を含む「リクエスト」を送信します。

3.

WEBサーバーでは、リクエストの内容を解釈して、APサーバーの「購入処理プログラム」へデータの更新や取得と、購入完了ページの加工を指示するリクエストを送信します。

4.

APサーバーでは、DBサーバーへ「商品ID=BBBBBの在庫を1つ減らす」「ユーザーID=AAAAAの残高を10,000円減らす」というデータ更新と、「商品ID=BBBBBの在庫」「ユーザーID=AAAAAの残高」の更新後データの取得を指示するリクエストを送信します。

5.

DBサーバーからAPサーバーへ、データ更新が成功したかどうかと、「商品ID=BBBBBの在庫」と「ユーザーID=AAAAAの残高」を「レスポンス」として送信します。

6.

APサーバーでは、DBサーバーからの情報を元に購入完了ページをHTML形式へ加工して、WEBサーバーへ

レスポンスとして送信します。

7.

WEBサーバーでは、APサーバーからの購入完了ページのHTMLを受け取り、ブラウザからのリクエストに応じた情報を付加して、ブラウザへレスポンスとして送信します。

8.

ブラウザでは、受け取ったHTMLを解釈して画面に表示させます。

1-9 「WEBアプリケーション」に必要なプログラム

ここまでで、これから作る「WEBアプリケーション」やそれを支える仕組みについて理解してきました。しかしまだ、肝心のJAVAがどこでどう動いているかがわかりません。最後に、1-8の図のどこでどのプログラムが動いているのかを学んでおきましょう。

1. ブラウザではHTML / css / javascript が動く

1-6において「ブラウザではHTMLを受信して表示する」と説明しましたが、内部では以下の3つが連携して画面表示や動きを制御しています。これらは今後課題の中で触れることになりますので覚えておきましょう。

(1) HTML：

HTML（HypeText Markup Language）には、画面の構造、表示する文字や画像、リンクやボタンを押した時に表示するURLなどが定義されており、ブラウザはこれを解釈して画面へ表示します。HTMLは本来、文章に「タイトル」や「見出し」といった構造を付加するものであり、装飾の機能はありません。

例）<h2>見出しレベル２</h2> ⇒ 文章の中見出しであることを示す。

(2) css：

css（Cascading Style Sheets）には、HTMLのタグや、idやclassにより識別される各要素に対して、色や大きさなどの表示方法について定義することができます。

例）h2 { color: red; } ⇒ 中見出しは赤字で表示させる。

(3) javascript：

ブラウザで動くプログラムです。身近な例では、申込フォームなどで必須入力項目が入力されるまで送信ボタンを押せなくしたり、送信ボタンを押した際に必須入力項目が未入力である旨メッセージを表示するといった処理によく使われます。

例）if (document.application_form.name.value === "") { alert('必須です！'); }

2. WEBサーバーではHTTP通信処理アプリケーションが動く

1-8では、WEBサーバーは情報を右から左へ流しているだけであまり活躍していないように見えますが、実はHTTP（HyperText Transfer Protocol）という約束に則ってブラウザやWEB上の他のサーバー達とやり取りを行う、とても重要な役割を担っています。HTTP通信処理アプリケーションにはApacheやnginxなどがあり、今後の課題ではApacheを使い、それに対する設定を行うことになります。

3. APサーバーではJAVAやPHPが動く

APサーバーは「アプリケーションサーバー」の略であり、WEBアプリケーションの中核処理を行うためのJAVAやPHPなどのプログラムと、それを実行するための処理アプリケーションが動いています。

今後の課題では、主にここで動かすJAVAのプログラムを書いていくこととなります。

4. DBサーバーではデータベース管理アプリケーションが動く

DB（Data Base）とは、アプリケーションに必要なデータを効率的に格納したり、取り出したりできる仕組みを示します。電話帳のように氏名と住所と電話番号を紐づけて管理するものをイメージしてください。

DBには様々な種類があり、表のような形式で扱えるRDB（Relational DB）が一般的で、それを管理するアプリケーションをRDBMS（RDB Management System）と言います。RDBMSにはAccess、Oracle、PostgreSQL、MySQL、SQLiteといったアプリケーションが存在します。

DBからデータを取り出したり、DBのデータを更新するためには、SQL（Simple Query Language）という言語を使用します。今後の課題ではSQLiteやMySQLを設定し、JAVAからSQLを発行して処理を行います。
